### Описание endpoint:
#### Client manager
- `GET /api/clients` : Возвращает список данных всех клиентов ({имя клиента} в формате json), сохраненных в БД.
- `GET /api/clients/{id}` : Возвращает данные клиента ({имя клиента} в формате json) по заданному идентификатору (id). 
- `POST /api/clients` : Сохраняет имя клиента в БД ({имя клиента} в теле запроса, в формате json)
#### Contact manager
- `GET /api/contacts/{id}` : Возвращает список контактов({тип,значение} в формате json) по заданному идентификатору клиента (id)
- `GET /api/contacts/{id}?type=(email/phone)` : Возвращает список контактов({тип,значение} в формате json) по заданному идентификатору клиента и указанному типу контакта (email/phone).
- `POST /api/contacts` : Сохраняет контакт ({id клиента,тип, значение}) в БД ({id клиента,тип, значение} в теле запроса, в формате json) 
####  OpenApi manager
- GET /openapi.yaml - Возвращает файл спецификации OpenAPI (в формате YAML). 
 
### В проекте используются библиотеки:
- Spring Boot Starter Web для разработки веб-приложений.
- Springdoc OpenAPI UI для автоматической генерации и документирования API с помощью OpenAPI.
- Java Persistence API (JPA) для управления взаимодействием с базой данных.
- Lombok для упрощения создания Java-классов с автоматической генерацией методов доступа и другими функциями.
- PostgreSQL в качестве базы данных.
- Liquibase Core для управления миграциями базы данных.
- MapStruct для автоматической генерации кода маппинга.

### Описание проекта:
#### Имя клиента:
- хранится в таблице clients
- имеет поля `id`,  `name`
- поле 'name' должно быть уникально
- для передачи данных имени клиента используется DTO класс `ClientNameDTO`

#### Контакт клиента:
- хранится в таблице contacts
- имеет поля `id`, `client_id` , `type`, `value`,
- поле `client_id` свзано с таблицей `clients` по полю `id`
- поле `type`  может содержать только значения, определенные в перечислении (enum) `ContactType`.
- совокупность значений `client_id` , `type`, `value` должна быть уникальной
- для передачи данных контакта клиента используются DTO классы `ContactDTO`, `ContactWithIdDTO` 

#### Контроллеры:
- `ClientController`  обрабатывает запросы для получения и сохранения информации о клиентах.
- `ContactController` отвечает за обработку запросов, связанных с контактами клиентов, включая получение и сохранение
  контактной информации
- `OpenApiController` обрабатывает запросы на предоставление спецификации

#### Сервисы:
- имеют два интерфейса и их соответствующие реализации.

#### Репозирии:
- включают два класса, которые расширяют JpaRepository, для таблиц `clients` и `contacts`.

#### Исключения:
- часть обрабатывается  `GlobalExceptionHandler` , для информирования *"фронта"* об ошибках в запросах

#### Логирование:
- путь для сохранения логов, формат, уровень логирования в `application.properties`  


#### docker-compose 
- для создания и запуска БД из контейнера
